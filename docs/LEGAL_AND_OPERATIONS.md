# @fan 출시 전 고려사항

이 문서는 `refer/` 자료를 바탕으로 @fan을 실제 서비스로 만들 때 확인해야 할 기술, 법무, 세무, 운영 이슈를 정리한 메모입니다. 법률·세무 자문이 아니며, 실제 결제/정산/인스타 연동 전에는 변호사, 세무사, PG사와 별도 확인이 필요합니다.

## 1. 정적 서버와 동적 서버

현재 MVP는 정적 웹앱입니다. 화면 검증, 발표, 클릭 가능한 프로토타입에는 충분합니다.

실서비스에는 동적 백엔드가 필요합니다.

- 회원가입, 로그인, 세션, 권한 관리
- 아티스트 본인 인증, 계정 수령, KYC
- 인스타그램 API 연동, 데이터 동의, 토큰 관리
- 결제 생성, PG 웹훅 검증, 환불, 취소, 정산 상태
- 1:1 채팅 저장, 실시간 전송, 신고/차단, 메시지 감사 로그
- 개인정보 열람/정정/삭제 요청 처리
- 운영자 어드민, 이상거래 탐지, CS 기록
- 세금계산서, 원천징수, 지급명세서, 정산 리포트

권장 단계:

1. 지금은 정적 프로토타입으로 유지
2. 실제 결제 전 `API 서버 + DB + 인증 + PG sandbox` 추가
3. 실제 인스타 연동 전 Meta 공식 API와 권한 검토
4. 실제 정산 전 PG/세무/약관 검토 완료

## 2. 인스타 ID로 아티스트 페이지 만들기

핵심 위험은 “공개된 인스타 ID니까 마음대로 써도 된다”가 아니라는 점입니다.

고려할 것:

- 인스타 ID가 특정 개인을 식별할 수 있으면 개인정보에 해당할 수 있습니다. 개인정보보호법은 성명, 영상 등으로 개인을 알아볼 수 있는 정보뿐 아니라 다른 정보와 쉽게 결합해 알아볼 수 있는 정보도 개인정보로 봅니다.
- 공개 프로필이라고 해도 수집·저장·검색 노출·후원 유도에 쓰려면 처리 목적, 보관 기간, 삭제 요청 경로, 법적 근거를 정해야 합니다.
- 아티스트가 가입하지 않았는데 페이지가 만들어지면 사칭, 혼동, 퍼블리시티권/성명권/초상권, 부정경쟁방지법상 식별표지 사용 문제가 생길 수 있습니다.
- 프로필 사진, 게시물 이미지, 소개글을 무단 복제하면 저작권/초상권/플랫폼 약관 문제가 커집니다.
- “후원금이 쌓였다”는 표현이 아티스트의 동의나 보증처럼 보이면 오인 가능성이 있습니다.

MVP 안전장치:

- 미수령 계정에는 “아티스트가 아직 수령하지 않은 비공식 팬 후원장” 표시
- 실제 인스타 사진/게시물 복제 금지
- 팬이 입력한 ID와 팬 메시지만 보관
- 아티스트 인증 전 정산, 전용 피드, 채팅 비활성
- 아티스트 삭제 요청, 소유권 주장, 신고 버튼 제공
- 악성/명예훼손 메시지 신고 및 숨김 처리
- 미수령 상태가 일정 기간 지속되면 자동 환불 또는 자동 비공개

실서비스에서는 Meta Platform Terms, Developer Policies, Instagram Platform 문서를 기준으로 공식 API 권한 범위 안에서만 조회/메시징해야 합니다. 크롤링이나 비공식 자동 DM은 피하는 쪽이 안전합니다.

## 3. 돈, PG, 예치, 정산

@fan은 “팬 → 플랫폼 → 아티스트”로 돈이 흐르는 순간 결제/정산 규제가 핵심이 됩니다.

전자금융거래법상 전자지급결제대행은 전자적 방법으로 결제정보를 송수신하거나 대가 정산을 대행/매개하는 것으로 정의됩니다. 금융위원회도 결제대금 정산에 관여하는 미등록 PG 개입, 미정산, 환불 애로, 거래 불투명성을 위험으로 설명합니다.

실무 선택지:

1. 등록 PG사의 일반 결제만 사용  
   단순 결제는 쉽지만, 플랫폼이 아티스트에게 따로 정산하면 내부 정산 리스크가 남습니다.

2. 등록 PG사의 마켓플레이스/파트너 정산 기능 사용  
   아티스트를 하위 판매자 또는 수취인으로 등록하고, PG가 정산/환불/정산 리포트를 지원하는 구조입니다. 초기 서비스에는 이 방향이 가장 현실적입니다.

3. 직접 PG 또는 관련 전자금융업 등록 검토  
   자본금, 내부통제, 정산자금 보호, 감독 대응 부담이 커서 초기 MVP에는 부적합합니다.

출시 전 필요한 것:

- PG사 계약
- 대표가맹점/하위가맹점 구조 검토
- 결제 취소, 부분환불, 충전 취소, 미수령 자동환불 정책
- 고객 돈을 회사 운영자금과 섞지 않는 구조
- 정산 지연, 정산 실패, 아티스트 미인증 시 처리
- 이상거래, 도난카드, 미성년자 결제, 차지백 대응
- 후원이 기부인지, 콘텐츠/리워드 구매인지 명확한 표시

주의: 법정 기부금 단체가 아니라면 “기부금 영수증” 같은 표현은 피하고, “후원”, “리워드”, “콘텐츠 이용” 등 실제 거래 성격에 맞춰 표시해야 합니다.

## 4. 사업자, 법인, 세금

국세청은 독립적으로 재화 또는 용역을 공급하는 사업자는 사업자등록을 해야 하고, 사업 개시일부터 20일 이내 신청해야 한다고 안내합니다. 사업 개시 전 신청도 가능합니다.

개인사업자와 법인:

- 개인사업자: 빠르게 시작 가능, 비용 낮음. 단, 결제/정산/책임이 대표 개인에게 귀속됩니다.
- 법인: 계약, 투자, 지분, 책임 분리, PG 심사, 정산 리스크 관리에 유리합니다. 실제 결제/정산을 다룰 계획이면 법인을 진지하게 검토해야 합니다.

세금 체크:

- 플랫폼 수수료 10%는 매출로 잡힐 가능성이 큽니다.
- 일반과세자는 매출세액에서 매입세액을 빼 부가가치세를 계산합니다.
- 아티스트가 개인인지 개인사업자인지 법인인지에 따라 원천징수, 세금계산서, 현금영수증, 지급명세서가 달라집니다.
- 국내 개인에게 사업소득 또는 기타소득을 지급하는 구조라면 원천징수와 지급명세서 제출 여부를 세무사와 확인해야 합니다.
- PG 수수료, 플랫폼 수수료, 아티스트 정산금, 환불금을 회계상 분리해야 합니다.

## 5. 통신판매, 약관, 개인정보

정부24는 통신판매업 신고를 전기통신매체 등을 통해 소비자와 직접 상거래가 이루어지는 경우의 민원으로 설명합니다. @fan이 후원 리워드, 전용 콘텐츠, 채팅권을 판매한다면 통신판매업 신고와 전자상거래법상 표시/광고, 청약철회, 환불 정책 검토가 필요할 수 있습니다.

출시 전 문서:

- 이용약관
- 유료서비스 약관
- 개인정보처리방침
- 아티스트 정산 약관
- 환불/취소 정책
- 커뮤니티 및 채팅 운영정책
- 신고/차단/권리침해 처리정책
- 미성년자 결제 및 법정대리인 동의 정책

## 6. 1:1 채팅 리워드

1:1 채팅은 매력적이지만 운영 리스크도 큽니다.

- 아티스트에게 과도한 응답 의무를 지우면 부담과 분쟁이 생깁니다.
- 팬 입장에서는 “돈으로 대화 구매”처럼 보일 수 있습니다.
- 성희롱, 스토킹, 악성 메시지, 개인정보 요구, 외부 연락처 요구를 막아야 합니다.
- 채팅 로그 보관 기간, 신고 시 열람 범위, 운영자 접근 권한이 필요합니다.

권장 정책:

- 100,000원 이상 후원 시 “3회 응답 보장” 같은 MVP 기준은 명확히 표시
- 답장 가능 기간과 불이행 시 환불 기준 명시
- 아티스트가 거절할 수 있는 권리 보장
- 팬의 무리한 요구는 신고/차단 가능
- 외부 연락처, 금전 요구, 민감정보 요청 금지

## 7. 출시 전 No-Go

아래 항목이 해결되기 전에는 실제 배포/결제를 열지 않는 것이 안전합니다.

- 실제 인스타 계정 조회/크롤링
- 미수령 아티스트의 사진/게시글 복제
- 실제 결제 버튼
- 회사 계좌로 후원금 직접 입금
- 아티스트 본인 인증 없는 정산
- 개인정보처리방침 없는 회원가입
- 환불 정책 없는 유료 리워드
- 신고/삭제 요청 없는 미수령 페이지 공개
